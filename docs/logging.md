# Observational logging

Append-only JSONL event stream capturing everything significant that happens in a running samsinn instance. Designed for:

- **Control-room studies** — multi-agent advisory tools used during real operations; retrospective analysis of operator ↔ agent ↔ tool dynamics.
- **Usage audits** — who used what, when, and what did the agents say.
- **Debugging production issues** — "what did the agent do right before it failed?"
- **Compliance / retention** — append-only record of conversations for regulated domains.
- **Mining training data** — finding good/bad interaction examples.

Different from the experiment runner (which writes per-run JSONs for controlled comparisons). Both can be active at once; they don't interfere.

> **Privacy notice.** Log files contain full message content, including anything typed by operators and generated by agents. This may include PII, sensitive domain data, or proprietary information. **The deployment is responsible for consent, retention, redaction, and access control** — samsinn itself doesn't encrypt, filter, or rotate by age. Treat log directories the same way you'd treat a database containing user conversations.

## Turn it on

Three ways — all equivalent.

### At boot (env vars)

```bash
SAMSINN_LOG_ENABLED=1 \
SAMSINN_LOG_DIR=/var/samsinn/logs \
SAMSINN_SESSION_ID=study-A-operator-1 \
  bun run start
```

### At runtime (REST)

```bash
# Start logging with a specific session id
curl -X PUT http://localhost:3000/api/logging \
  -H 'Content-Type: application/json' \
  -d '{"enabled":true,"sessionId":"shift-morning-1"}'

# Swap to a new session (without stopping samsinn)
curl -X PUT http://localhost:3000/api/logging \
  -H 'Content-Type: application/json' \
  -d '{"sessionId":"shift-afternoon-1"}'

# Change directory
curl -X PUT http://localhost:3000/api/logging \
  -H 'Content-Type: application/json' \
  -d '{"dir":"/mnt/external-drive/samsinn-logs"}'

# Stop
curl -X PUT http://localhost:3000/api/logging \
  -H 'Content-Type: application/json' \
  -d '{"enabled":false}'

# Inspect state + stats
curl http://localhost:3000/api/logging
```

### At runtime (MCP)

The MCP tool surface mirrors the REST API — useful for agents that operate samsinn headlessly:

```
configure_logging({enabled: true, sessionId: "my-study", kinds: ["message.*","agent.*"]})
get_logging()  →  {enabled, dir, sessionId, kinds, currentFile, stats: {eventCount, droppedCount, ...}}
```

## Config reference

| Control | Env var | REST field | Default |
|---|---|---|---|
| On/off | `SAMSINN_LOG_ENABLED=1` | `enabled: true` | `false` |
| Output directory | `SAMSINN_LOG_DIR` | `dir` | `~/.samsinn/logs/` |
| Session identifier | `SAMSINN_SESSION_ID` | `sessionId` | `session-<ts>-<shortId>` |
| Kind filter | `SAMSINN_LOG_KINDS` (comma-separated) | `kinds: string[]` | `["*"]` |

Changing `sessionId` at runtime closes the current file (with `session.end`) and opens a new one (with `session.start`). This is the primary way to bracket operator shifts, task boundaries, etc.

Changing `dir` at runtime behaves the same way — old file closes, new file opens in the new directory.

## File layout

```
<dir>/
  <sessionId>.jsonl       ← first file
  <sessionId>.1.jsonl     ← rotated when first exceeds 100 MB
  <sessionId>.2.jsonl
  ...
```

Each line is one JSON object:

```json
{"ts":1777040000000,"kind":"message.posted","session":"shift-morning-1","roomId":"...","actor":{"kind":"ai","id":"...","name":"Advisor"},"payload":{"message":{"content":"...","type":"chat","promptTokens":47,"completionTokens":12,"toolTrace":[...]}}}
```

Fields:

- `ts` — event timestamp (ms epoch)
- `kind` — dotted event type (see kinds below)
- `session` — the session identifier active when the event was emitted
- `actor` — `{kind: 'human'|'ai'|'system'|'researcher'|'unknown', id?, name?}` — best-effort attribution
- `roomId` — present for room-scoped events
- `payload` — event-specific; always an object

## Event kinds

| Kind | Fires when |
|---|---|
| `session.start` | Logging activated — always first line of each file |
| `session.end` | Logging deactivated / session swapped / samsinn shutdown |
| `room.created` | A room is created |
| `room.deleted` | A room is deleted |
| `room.membership_changed` | Agent added to / removed from a room |
| `room.delivery_mode_changed` | Room delivery mode changed (broadcast / macro / …) |
| `room.mode_auto_switched` | Mode auto-changed (e.g. second AI joined) |
| `message.posted` | Any message — includes full content, telemetry, toolTrace |
| `agent.eval_event` | Tool start / tool result / warning during agent evaluation |
| `artifact.changed` | Artifact added / updated / removed / resolved |
| `macro.event` | Macro started / step completed / completed / cancelled |
| `provider.bound` | LLM provider bound to an agent's call |
| `provider.all_failed` | All providers failed for a model |
| `provider.stream_failed` | Provider streaming error |
| `summary.config_changed` | Room summary/compression config changed |
| `summary.updated` | Summary/compression produced for a room |
| `summary.run_started` | Summary generation started |
| `summary.run_completed` | Summary generation completed |
| `summary.run_failed` | Summary generation failed |
| `log.dropped` | Synthetic — logged when events were dropped due to queue overflow |
| `log.serialize_failed` | Synthetic — logged when an event couldn't be JSON.stringified |

Deliberately not logged (noise):

- `turn.changed` — fires many times per eval
- `summary.run_delta` — per-chunk tokens during streaming
- UI state events (`bookmarks.changed`, `macro_selection.changed`)

Filter with `kinds` to restrict:

```bash
SAMSINN_LOG_KINDS='message.*,agent.*'   # just conversation + tool use
SAMSINN_LOG_KINDS='*'                    # everything (default)
SAMSINN_LOG_KINDS='session.*,room.*'    # session boundaries only
```

Glob forms: `*` matches all; `prefix.*` matches anything starting with `prefix.`; exact strings match exactly; comma-separated = union.

## Analysis examples

### jq — count events by kind

```bash
jq -r '.kind' shift-morning-1.jsonl | sort | uniq -c | sort -rn
```

### jq — every message from a specific agent

```bash
jq -c 'select(.kind=="message.posted" and .actor.name=="Advisor") | .payload.message.content' shift-morning-1.jsonl
```

### jq — tool-use frequency

```bash
jq -c 'select(.kind=="message.posted" and .payload.message.toolTrace) | .payload.message.toolTrace[].tool' shift-morning-1.jsonl | sort | uniq -c | sort -rn
```

### Python / pandas — messages as a DataFrame

```python
import pandas as pd

def messages(path):
    rows = []
    for line in open(path):
        e = json.loads(line)
        if e["kind"] == "message.posted":
            m = e["payload"]["message"]
            rows.append({
                "ts": e["ts"],
                "sender": m.get("senderName"),
                "model": m.get("model"),
                "tokens_in": m.get("promptTokens"),
                "tokens_out": m.get("completionTokens"),
                "tool_count": len(m.get("toolTrace") or []),
                "content": m.get("content"),
            })
    return pd.DataFrame(rows)

df = messages("shift-morning-1.jsonl")
df.groupby("sender")[["tokens_in","tokens_out","tool_count"]].sum()
```

### DuckDB — direct JSONL query

```sql
SELECT kind, COUNT(*) AS n
FROM read_json_auto('shift-*.jsonl', format='newline_delimited')
GROUP BY kind
ORDER BY n DESC;
```

## Robustness behavior

- **Disk full / permission denied**: sink catches the error, writes one `[logging] sink write failed` line to stderr, drops that batch and increments the dropped counter. Samsinn keeps running. The dropped count surfaces in `GET /api/logging` for monitoring.
- **Queue overflow** (>10k events backed up): sink drops the oldest events, emits a `log.dropped` synthetic event on the next successful write so analysts see the gap, and logs a stderr warning. This is rare in practice — the 1s flush interval clears the queue quickly.
- **Observer throws**: one observer misbehaving doesn't kill the others or samsinn. Errors surface on stderr.
- **Session swap mid-write**: the swap drains the current sink cleanly before opening the new one — no events lost across a reconfigure.
- **Two samsinn processes with the same `sessionId`**: they append to the same file, interleaved. Distinct deployments = distinct session ids.

## Retention + rotation — deployment responsibility

Samsinn rotates files when they exceed 100 MB. It does NOT:

- Delete old files (run logrotate, a cron job, or a research-data archival pipeline).
- Compress rotated files (pipe them through gzip/zstd in your archival step).
- Redact content (content redaction is a follow-up feature — for now, control access to the log directory).
- Enforce retention (GDPR, institutional IRB, etc. — apply at the deployment layer).

## Performance

- **Write path**: non-blocking — `sink.write()` is a synchronous enqueue, the background flush loop writes to disk every 1s.
- **Typical overhead per event**: sub-microsecond enqueue + a JSON.stringify at flush time. Negligible.
- **Flush I/O**: one `appendFile` per second per active sink (batch-mode).
- **Memory**: bounded at 10k events in-memory. One session.start + 10k average events ≈ a few MB.
